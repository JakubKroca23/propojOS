# Plan 1.3: Appwrite Integration & Auth

## Objective
Initialize Appwrite SDK and build the basic authentication UI.

## Context
- .gsd/SPEC.md
- .gsd/DECISIONS.md
- src/App.tsx

## Tasks

<task type="auto">
  <name>Initialize Appwrite SDK</name>
  <files>package.json, .env, src/lib/appwrite.ts</files>
  <action>
    - Install the `appwrite` SDK (`npm install appwrite`).
    - Create a `.env` file with `VITE_APPWRITE_ENDPOINT=https://appwrite.propoj.app/v1` and `VITE_APPWRITE_PROJECT=propoj-os`.
    - Create `src/lib/appwrite.ts` and initialize the Appwrite Client and Account instances using the environment variables.
    - Export the `client` and `account` instances.
  </action>
  <verify>test -f src/lib/appwrite.ts && grep "Client" src/lib/appwrite.ts</verify>
  <done>Appwrite SDK is installed and configured with the provided credentials.</done>
</task>

<task type="auto">
  <name>Build Login UI and Auth Logic</name>
  <files>src/pages/Login.tsx, src/pages/Login.module.css, src/App.tsx</files>
  <action>
    - Create `src/pages/Login.tsx` with a simple email/password form.
    - Implement login logic using `account.createEmailPasswordSession()`.
    - Handle success and error states.
    - Create `src/pages/Login.module.css` (or regular css) to style the form using Vanilla CSS.
    - Update `src/App.tsx` to conditionally render the Login page or the "Welcome" dashboard based on authentication state (check session on mount).
  </action>
  <verify>test -f src/pages/Login.tsx</verify>
  <done>Login page is created and wired to Appwrite Auth.</done>
</task>

## Success Criteria
- [ ] Appwrite client is initialized.
- [ ] Login UI exists and attempts authentication against Appwrite.
