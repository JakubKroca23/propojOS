# Plan 1.1: Docker & Traefik Infrastructure

## Objective
Set up the local Docker and Traefik infrastructure.

## Context
- .gsd/SPEC.md
- .gsd/DECISIONS.md

## Tasks

<task type="auto">
  <name>Create docker-compose.yml</name>
  <files>docker-compose.yml</files>
  <action>
    Create a `docker-compose.yml` file.
    - Add a `traefik` service (image: traefik:v3.0, command: --api.insecure=true, --providers.docker=true, --entryPoints.web.address=:80).
    - Add a `web` service (build: ., ports: 5173:5173 for local dev without traefik if needed).
    - Configure Traefik labels on the `web` service to route `propoj.localhost` to the `web` service on port 5173.
    - Connect the `web` service to the existing `appwrite` external network.
  </action>
  <verify>docker compose config -q</verify>
  <done>docker-compose.yml is valid and contains traefik and web services with the correct labels and network.</done>
</task>

<task type="auto">
  <name>Create development Dockerfile</name>
  <files>Dockerfile</files>
  <action>
    Create a `Dockerfile` for the Vite/React application.
    - Use `node:20-alpine` as the base image.
    - Set `WORKDIR /app`.
    - Copy package files and run `npm install`.
    - Copy the rest of the code.
    - Expose port 5173.
    - CMD should be `npm run dev -- --host`.
  </action>
  <verify>test -f Dockerfile</verify>
  <done>Dockerfile is created with correct instructions to run the Vite dev server.</done>
</task>

## Success Criteria
- [ ] docker-compose.yml exists and is valid.
- [ ] Dockerfile exists and is valid.
