# Plan 3.1: Vite Module Federation Setup

## Objective
Configure Vite to support Module Federation, enabling the PropojOS host to load remote modules dynamically.

## Context
- .gsd/SPEC.md
- vite.config.ts

## Tasks

<task type="auto">
  <name>Install and Configure Federation Plugin</name>
  <files>vite.config.ts</files>
  <action>
    - Install `@originjs/vite-plugin-federation`.
    - Update `vite.config.ts` to include the federation plugin.
    - Configure the host to share dependencies like `react`, `react-dom`, and `zustand` so external apps don't bundle them.
    - Prepare the configuration to support dynamic remotes (e.g., using `remotes` config with a resolver, or just setting up the shared dependencies if remotes are loaded dynamically via JS script injection).
  </action>
  <verify>test -f vite.config.ts && npx tsc</verify>
  <done>Vite config is updated with federation and compiles cleanly.</done>
</task>

## Success Criteria
- [ ] `@originjs/vite-plugin-federation` is installed.
- [ ] Shared dependencies are properly configured in `vite.config.ts`.
