# Plan 4.1: The Event Bus

## Objective
Establish a global event bus that allows loosely coupled micro-apps to communicate in real-time.

## Context
- .gsd/SPEC.md
- src/store/osStore.ts

## Tasks

<task type="auto">
  <name>Create Event Store</name>
  <files>src/store/eventStore.ts</files>
  <action>
    - Create a new Zustand store `eventStore.ts`.
    - It should manage a list of `connections` (sourceId, sourcePort -> targetId, targetPort).
    - It should expose an `emit(sourceId, port, data)` function.
    - When `emit` is called, it finds all connections from that source port, and updates a state map of `inputs[targetId][targetPort] = data`.
  </action>
  <verify>test -f src/store/eventStore.ts</verify>
  <done>Zustand store for events is fully structured.</done>
</task>

<task type="auto">
  <name>Create App Hooks</name>
  <files>src/hooks/useAppEvent.ts</files>
  <action>
    - Create a React Hook that plugins can use to interact with the Event Bus.
    - `useAppInput(appId, portName)`: returns the latest data piped into this port.
    - `useAppOutput(appId, portName)`: returns an `emit(data)` function to send data out.
  </action>
  <verify>test -f src/hooks/useAppEvent.ts</verify>
  <done>Hooks are ready for plugins to consume.</done>
</task>

## Success Criteria
- [ ] Apps can emit data to output ports.
- [ ] Apps can receive data from input ports via Zustand reactivity.
